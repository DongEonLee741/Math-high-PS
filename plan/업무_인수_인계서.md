## 1. 프로젝트 개요

고등학교 확률과 통계 수업용 웹 앱으로, **학습지**(학생용)와 **슬라이드**(교사용)를 제공합니다.
- 학생은 iPad + Apple Pencil로 학습지에 필기하고 Google Drive에 저장
- 교사는 슬라이드로 수업하며 학생 현황을 모니터링
- Firebase Hosting으로 배포, GitHub Actions CI/CD 자동화

## 3. 기술 스택

## 4. 인증 시스템 (index.html) — 현재 상태

### 4.1 인증 흐름

```
index.html 접속
  → 로그인 오버레이 표시
  → Google 로그인 버튼 클릭
  → Firebase Auth (signInWithPopup)
  → 이메일로 역할 판별 (roal2745@gmail.com = 교사)
  → sessionStorage에 인증 정보 저장
  → Apps Script로 redirect (Drive OAuth)
  → 돌아올 때 ?driveEmail= 파라미터
  → sessionStorage에 driveSync_email 저장
  → 역할별 UI 표시
  → 학습지 클릭 시 ?driveEmail= & ?class= 자동 전달
```

### 4.2 역할 기반 UI

| 역할 | 보이는 콘텐츠 | CSS 메커니즘 |
|------|-------------|-------------|
| 학생 (기본) | 학습지만 | `.link-slide { display: none; }` |
| 교사 | 학습지 + 슬라이드 | `body.role-teacher .link-slide { display: inline-flex; }` |

## 5. Google Drive 저장 시스템

### 5.1 계층 폴더 구조 (신규)

학습지 데이터가 Drive에 저장될 때 아래 구조로 생성됩니다:

```
내 드라이브/
└── 확률과통계/
    └── 1단원_순열과조합/
        ├── 2차시_중복순열.json
        ├── 3차시_같은것이있는순열.json
        ├── 4차시_스스로확인하기.json
        ├── 6차시_중복조합.json
        ├── 7차시_스스로확인하기_중복조합.json
        └── 9차시_이항정리.json
```

### 5.2 학습지 파일의 키 구조

각 학습지 HTML에는 두 개의 키 함수가 있습니다:

```javascript
// localStorage 용 (기존 호환)
function getStorageKey() {
    return 'worksheet_PS_2차시_중복순열';
}

// Drive 저장용 (계층 경로)
function getDriveKey() {
    return '확률과통계/1단원_순열과조합/2차시_중복순열';
}
```

### 5.3 Apps Script (Code.gs) — 핵심 함수

| 함수 | 설명 |
|------|------|
| `parseKey(key)` | `'확률과통계/1단원/2차시'` → `{folderPath: [...], fileName: '2차시'}` |
| `getOrCreateFolderByPath(parts)` | 중첩 폴더 생성/탐색 |
| `doGet(e)` | `?action=login/profile/list/load/callback` 처리 |
| `doPost(e)` | `{"action":"save", "key":"...", "data":{...}}` 처리 |

## 6. 배포 구조

### 6.1 GitHub Actions (자동)

```
git push origin main
  → .github/workflows/firebase-hosting.yml 실행
  → Firebase Hosting에 자동 배포 (~35초)
```

### 6.2 firebase.json 주요 설정

```json
{
  "hosting": {
    "public": ".",
    "ignore": ["firebase.json", "node_modules", "**/*.pdf", "**/*.md", "plan/**", ...]
  }
}
```

### 6.3 Google Cloud Console OAuth 설정

| 항목 | 값 |
|------|---|
| 프로젝트 | math-high-cm1 |
| OAuth 클라이언트 | Web client (auto created by Google Service) |
| 승인된 JS 원본 | `http://localhost`, `http://localhost:5000`, `https://math-class.it.kr` |
| 승인된 리디렉션 URI | `https://math-class.it.kr/__/auth/handler` |

## 11. 주의사항

- `authDomain`은 반드시 `math-class.it.kr` 사용 (`.web.app` 사용 시 redirect_uri_mismatch)
- `sessionStorage` 사용 → 탭 닫으면 로그아웃 (공유 iPad 대응)
- `TEACHER_EMAILS` 배열에 교사 이메일 추가 가능 (현재 `roal2745@gmail.com`만)
- firebase.json의 ignore에 `plan/**`이 포함 → plan 폴더는 배포 안 됨
- 학습지의 `getStorageKey()`는 localStorage용, `getDriveKey()`는 Drive 저장용 (분리됨)
