## 4. 정답표

### Phase 1: 교과서 문제

**스스로 확인하기 1**: $\frac{1}{2}$

**스스로 확인하기 2**: $0.72369$
- 풀이: $\frac{144738}{200000} = 0.72369$

**스스로 확인하기 3**: $\frac{18}{35}$
- 풀이: $\frac{_{3}\mathrm{C}_{2} \times _{4}\mathrm{C}_{2}}{_{7}\mathrm{C}_{4}} = \frac{3 \times 6}{35} = \frac{18}{35}$

**스스로 확인하기 4**: $\frac{2}{5}$
- 풀이: 전체 삼각형 수 $_{6}\mathrm{C}_{3} = 20$, 이등변삼각형 수 = 8, $P = \frac{8}{20} = \frac{2}{5}$
- 이등변삼각형 목록 (꼭짓점 번호 1~6, 시계방향):
  - 정삼각형(2개): {1,3,5}, {2,4,6} — 호 간격 (2,2,2), 세 변 모두 $r\sqrt{3}$
  - 이등변삼각형(6개, 연속 3점): {1,2,3}, {2,3,4}, {3,4,5}, {4,5,6}, {5,6,1}, {6,1,2} — 호 간격 (1,1,4), 두 변 $r$ + 한 변 $r\sqrt{3}$
  - (※ 정삼각형도 이등변삼각형에 포함)
  - 나머지 12개는 부등변삼각형: 호 간격 (1,2,3), 세 변이 모두 다름 ($r, r\sqrt{3}, 2r$)

**스스로 확인하기 5**: $\frac{10}{21}$
- 풀이: 홀수 5개{1,3,5,7,9}, 짝수 4개{2,4,6,8}
  - 홀수 1개 + 짝수 2개: $_{5}\mathrm{C}_{1} \times _{4}\mathrm{C}_{2} = 5 \times 6 = 30$
  - 홀수 3개: $_{5}\mathrm{C}_{3} = 10$
  - 합: $30 + 10 = 40$
  - 전체: $_{9}\mathrm{C}_{3} = 84$
  - $P = \frac{40}{84} = \frac{10}{21}$

**수학으로 생각 넓히기**: $\frac{5}{27}$
- 풀이: 수빈(a), 지원(b), 선아(c), 조건 a < b < c 또는 c < b < a
  - b=2: a 또는 c 중 하나가 1, 나머지가 {3,4,5,6} → 1×4×2 = 8가지
  - b=3: 2×3×2 = 12가지
  - b=4: 3×2×2 = 12가지
  - b=5: 4×1×2 = 8가지
  - 합: 8+12+12+8 = 40
  - 전체: $6^3 = 216$
  - $P = \frac{40}{216} = \frac{5}{27}$

### Phase 2: 외부 문제

**문제 1**:
- (1) $S = \{1, 2, 3, 4, 5, 6\}$
- (2) $A = \{1, 3, 5\}$
- (3) $B = \{3, 6\}$

**문제 2** (A = {5,6}, B = {1,2,3,6}):
- (1) $A \cup B = \{1, 2, 3, 5, 6\}$
- (2) $A \cap B = \{6\}$
- (3) $A^C = \{1, 2, 3, 4\}$
- (4) $A^C \cup B^C = \{1, 2, 3, 4, 5\}$

**문제 3** (흰 공 5개 + 검은 공 3개에서 3개):
- (1) 흰 공 3개: $\frac{_{5}\mathrm{C}_{3}}{_{8}\mathrm{C}_{3}} = \frac{10}{56} = \frac{5}{28}$
- (2) 흰 공 1개 + 검은 공 2개: $\frac{_{5}\mathrm{C}_{1} \times _{3}\mathrm{C}_{2}}{_{8}\mathrm{C}_{3}} = \frac{15}{56}$

**문제 4** (서로 다른 두 주사위):
- (1) 두 눈이 같을 확률: $\frac{6}{36} = \frac{1}{6}$
- (2) 합이 12: $\frac{1}{36}$
- (3) 곱이 0: $\frac{0}{36} = 0$ (불가능 사건)

**문제 5**: ㄴ (A와 B는 서로 배반사건)
- ㄱ: A∪C=S는 보장되지 않음 (A∩C=∅이지만 A가 C의 여사건인지는 알 수 없음)
- ㄴ: B⊂C이고 A∩C=∅이므로 A∩B=∅ → 배반사건 ✓
- ㄷ: A∩C=∅이 A∪C=S를 보장하지 않으므로 여사건 아님

### 인터랙티브 ① 원 위의 이등변삼각형 탐색기

**삽입 위치**: 스스로 확인하기 4번 문제 텍스트 직후, 풀이/정답 공개 전

**Level**: Level 2 (교과서에 없는 새 인터랙티브 경험) — 결정 완료

**교육적 목적**: 학생이 원 위의 등간격 점에서 삼각형을 직접 만들어보며, "같은 호 길이 → 같은 변 길이 → 이등변"이라는 **수학적 구조를 직관으로 발견**한 뒤, 체계적 경우 세기로 넘어가는 흐름.

**핵심 설계 원칙**: 직관 먼저, 형식 나중 (비형식적 탐색 → 구조 발견 → 형식적 계산)

#### 인터랙션 흐름

**단계 1: 자유 탐색** (3~5개 삼각형 만들기)
- 원 위의 6개 점에 번호(①~⑥) 표시. 점 사이 간격이 동일함을 시각적으로 명확히
- 학생이 점 3개를 탭하면 삼각형이 SVG로 즉시 그려짐
- 세 변의 길이가 변 위에 숫자로 표시됨 (정규화된 값: 단변=1, 중변=√3, 장변=2)
- **같은 길이의 변은 같은 색으로 강조** (예: 단변=파랑, 중변=초록, 장변=주황)
- 이등변인지 아닌지에 대한 텍스트 판정 없음 — 학생이 색을 보고 스스로 판단
- "다른 삼각형 만들기" 버튼으로 초기화 후 새 조합 시도
- 화면 하단에 "만들어본 삼각형 목록"이 작은 아이콘으로 누적 표시 (최대 5개)

**단계 2: 구조 발견 유도** (3개 이상 만든 후 표시)
- 학생이 3개 이상의 삼각형을 만들면, 유도 질문이 자연스럽게 나타남:
  - "만들어본 삼각형들을 관찰해 보세요. 변의 길이가 몇 종류인가요?"
  - "같은 색 변이 2개 이상인 삼각형은 어떤 특징이 있나요?"
- 유도 질문은 텍스트로만 제시. 입력 요구 없음. **생각할 거리를 던지는 것**이 목적.

**단계 3: 전체 보기** (선택적)
- "모든 삼각형 보기" 버튼 → 20개 삼각형이 작은 격자로 한눈에 표시
- 이등변인 것 8개가 시각적으로 구별됨 (같은 색 변 2개 이상)
- 하단에 "$\frac{8}{20} = \frac{2}{5}$" 수식이 자연스럽게 도출
- **이 단계는 학생이 풀이를 시도한 뒤 확인용으로 사용** — 자동으로 열리지 않음

#### 수학적 정확성 요구사항

- 6개 점의 좌표: 정육각형 꼭짓점, 반지름 r 기준
  - 점 k (k=1~6): $(r \cos(90° - 60°(k-1)),\ r \sin(90° - 60°(k-1)))$
- 변의 길이 계산: 두 점 사이의 유클리드 거리
  - 인접 점 (호 1칸): $r$ (단변)
  - 호 2칸: $r\sqrt{3}$ (중변)  
  - 호 3칸 (지름): $2r$ (장변)
- 이등변 판별: 세 변 중 두 변 이상이 같은 길이 (epsilon 비교: `Math.abs(a-b) < 0.001`)
- 정삼각형({1,3,5}, {2,4,6})도 이등변에 포함

#### 피드백 설계

- ✓/✗, "정답", "틀림" 메시지 일절 없음
- 같은 길이의 변을 같은 색으로 표시하는 것이 유일한 피드백
- 학생이 색 패턴을 보고 "아, 이건 두 변이 같네" 또는 "세 변이 다 같네" 또는 "다 다르네"를 스스로 판단

**Level**: Level 2 (1차 학습지의 벤 다이어그램을 복습 맥락에서 확장) — 결정 완료

**교육적 목적**: 학생이 원소를 벤 다이어그램의 올바른 영역에 직접 배치하면서, 집합 연산($A \cup B$, $A \cap B$, $A^C$, $A^C \cup B^C$)의 **구조를 직관으로 파악**한 뒤, 형식적 집합 표기로 정리.

```
│   └──┘ └──┘ └──┘ └──┘ └──┘ └──┘        │
│                                           │
│  [벤 다이어그램 SVG]                       │
│   ┌──────────────────────────┐           │
│   │    ┌───A───┐  ┌───B───┐  │           │
│   │    │       │  │       │  │           │
│   │    │   A만 │A∩B│  B만  │  │           │
│   │    │       │  │       │  │           │
│   │    └───────┘  └───────┘  │           │
│   │         S 바깥 영역       │           │
│   └──────────────────────────┘           │
│                                           │
│  [확인 버튼]  [초기화 버튼]                 │
│                                           │
│  [집합 연산 질문 영역 — 배치 완료 후 표시]   │
└─────────────────────────────────────────┘
```

#### 인터랙션 흐름

**단계 1: 원소 배치** (직관적 구조 파악)
- 상단에 원소 카드 {1, 2, 3, 4, 5, 6}이 일렬로 나열
- 벤 다이어그램에 4개 영역이 시각적으로 구분: "A만", "A∩B", "B만", "A도B도 아닌 곳"
- A = {5, 6} (4보다 큰 눈), B = {1, 2, 3, 6} (6의 약수)가 다이어그램 상단에 텍스트로 표시
- 학생이 원소 카드를 **드래그하여** 벤 다이어그램의 적절한 영역에 배치 (Pointer Events)
- 6개 원소를 모두 배치하면 "확인" 버튼 활성화

**배치 결과 피드백**:
- "확인"을 누르면, 올바른 위치에 있는 원소는 **해당 영역이 살짝 밝아지고**, 잘못된 위치의 원소는 **원래 자리로 부드럽게 돌아감** (바운스 애니메이션 없이 슬라이드)
- "틀렸습니다" 메시지 없음. 돌아온 원소를 보고 학생이 "이건 여기가 아니구나"를 스스로 판단
- 올바른 배치가 완료되면 벤 다이어그램이 완성된 상태로 고정

**정답 배치**:
- A만 영역: {5}
- A∩B 영역: {6}
- B만 영역: {1, 2, 3}
- 바깥 영역: {4}

**단계 2: 집합 연산 탐색** (배치 완료 후 활성화)
- 4개 버튼이 나타남: "$A \cup B$", "$A \cap B$", "$A^C$", "$A^C \cup B^C$"
- 버튼을 누르면 해당 연산의 결과 영역만 강조(배경색 변경), 나머지 영역은 dim 처리
- 강조된 영역 내의 원소가 목록으로 하단에 표시됨
- 예: "$A \cup B$" 클릭 → A만 + A∩B + B만 영역 강조 → {1, 2, 3, 5, 6} 표시
- **KaTeX 수식이 실시간으로 연동**: "$A \cup B = \{1, 2, 3, 5, 6\}$"

**단계 3: 학생 직접 시도** (선택적 — 버튼 옆에 "직접 해보기" 토글)
- "직접 해보기" 모드를 켜면, 버튼 클릭 대신 학생이 영역을 직접 탭
- "$A^C$에 해당하는 영역을 탭해 보세요" → 학생이 영역 탭 → 탭한 영역이 강조 → "확인" → 올바른 영역과 비교
- 이 모드에서는 **원소 단위가 아닌 영역 단위**로 탭 (인터랙션 단순화)

#### 수학적 정확성 요구사항

- S = {1, 2, 3, 4, 5, 6}
- A = {5, 6} (4보다 큰 눈: 5, 6)
- B = {1, 2, 3, 6} (6의 약수: 1, 2, 3, 6)
- A ∪ B = {1, 2, 3, 5, 6}
- A ∩ B = {6}
- A^C = {1, 2, 3, 4}
- B^C = {4, 5}
- A^C ∪ B^C = {1, 2, 3, 4, 5}

#### 피드백 설계

- ✓/✗, "정답", "틀림" 메시지 없음
- 원소 배치: 잘못된 원소가 원래 위치로 되돌아가는 시각적 피드백만 제공
- 집합 연산: 영역 강조 + 원소 목록 + 수식 표시로 학생이 스스로 확인
- 드래그 시 원소 카드가 손가락/마우스를 따라 이동하며, 영역 위에 올리면 해당 영역이 살짝 하이라이트 (드롭 가능 피드백)

| 원칙 | 적용 |

- 인터랙티브가 있는 문제(4번, 외부 2번)는 인터랙티브 체험 후에 풀이 토글 사용 권장 (강제 아님)

## 8. 슬라이드 명세: 말판 게임 풀이 시각화 (대칭성 풀이)

> **산출물 형태**: 별도 단일 HTML 파일 (슬라이드 형식)
> **용도**: 교사가 "수학으로 생각 넓히기" 문제를 설명할 때 화면에 띄우는 시각 보조 자료
> **풀이 방식**: 교과서의 열거적 접근이 아닌, **대칭성을 이용한 구조적 접근** (교사 제공 풀이)

### 풀이의 핵심 논증 구조

1. 전체 경우의 수: $n(S) = 6^3 = 216$
2. 서로 다른 3개의 수가 나오는 경우의 수: $6 \times 5 \times 4 = 120$ (누가 던졌는지 구분 없이)
3. 이 120가지를 크기 순으로 정렬하면 □ < □ < □ — 가운데 자리에 올 사람은 3명 중 1명
4. 대칭성: 지원이가 가운데일 경우 = $\frac{6 \times 5 \times 4}{3} = 40$
5. 확률: $\frac{40}{216} = \frac{5}{27}$

### 슬라이드 내용

---

**슬라이드 1: 문제 상황 제시**

- 말판 (1~6칸) 위에 수빈(빨강), 지원(파랑), 선아(초록) 말 3개가 출발점에 위치
- "각자 주사위를 한 번씩 던져 나온 수만큼 이동"
- 구체적 예시 하나를 시각적으로 시연:
  - 수빈=2, 지원=4, 선아=6 → 말판 위에 말 이동 후 위치 표시
  - 말판 위 배치: 수빈(2칸) — 지원(4칸) — 선아(6칸)
  - "지원이가 수빈과 선아 사이에 있다!" 라고 상황 확인
- 하단에 질문: "이런 일이 일어날 확률은?"

---

**슬라이드 2: 전체 경우의 수**

- $n(S) = 6 \times 6 \times 6 = 6^3 = 216$
- 시각화: 주사위 3개를 나란히 배치, 각각 1~6 가능
  ```
  [수빈 🎲] × [지원 🎲] × [선아 🎲]
    6가지      6가지      6가지    = 216가지
  ```
- "이제 이 중에서 지원이가 가운데인 경우를 세야 합니다"

---

**슬라이드 3: 같은 수가 나오면? — 배제해야 하는 경우**

- 핵심 질문: "세 수 중에 같은 수가 있으면 '사이에 있다'가 성립할까?"
- 구체적 예시 3개를 말판 위에 시각화:
  - 수빈=3, 지원=3, 선아=5 → 말판: 수빈과 지원이 같은 위치
    - "수빈과 지원이 같은 자리 → 지원이가 '사이'에 있다고 할 수 없다"
  - 수빈=4, 지원=4, 선아=4 → 말판: 세 명 모두 같은 위치
    - "셋 다 같은 자리 → '사이'라는 개념 자체가 없다"
  - 수빈=2, 지원=5, 선아=5 → 말판: 선아와 지원이 같은 위치
    - "지원이가 선아와 같은 자리 → '사이'가 아니다"
- **결론 강조**: "지원이가 사이에 있으려면, **세 수가 모두 달라야** 합니다"
- 이 결론이 다음 슬라이드의 "서로 다른 수" 조건으로 자연스럽게 연결

---

**슬라이드 4: 서로 다른 3개의 수 — □□□ 배열**

- "누가 던졌는지 잊고, 나온 수만 크기 순으로 정렬해 봅시다"
- 시각화: 3개의 빈 칸이 크기 순으로 배치
  ```
  □ < □ < □
  작은 수  가운데  큰 수
  ```
- "서로 다른 3개의 수를 뽑는 경우의 수는?"
- 단계별 표시 (progressive disclosure):
  1. 첫 번째 칸: 6가지 (1~6 중 하나)
  2. 두 번째 칸: 5가지 (첫 번째를 제외)
  3. 세 번째 칸: 4가지 (첫째, 둘째를 제외)
  4. → $6 \times 5 \times 4 = 120$가지
- 구체적 예시: {1, 4, 6}을 뽑았다면 → 1 < 4 < 6으로 정렬

---

**슬라이드 5: 대칭성 — 가운데 자리는 누구 것?**

- 이 슬라이드가 풀이의 핵심. **시각적으로 가장 공을 들여야 하는 슬라이드**.

- **Step 1**: □ < □ < □ 배열에서 가운데 자리(□)를 강조
  ```
  □ < [  ?  ] < □
        ↑
    가운데 자리
  ```
  "서로 다른 3개의 수를 크기 순으로 놓으면, 가운데에 올 사람은 누구일까?"

- **Step 2**: 가운데 자리 후보는 3명 — 수빈, 지원, 선아
  ```
  □ < [수빈] < □    ← 수빈이가 가운데
  □ < [지원] < □    ← 지원이가 가운데  ★ 우리가 원하는 것
  □ < [선아] < □    ← 선아가 가운데
  ```
  "3명이 완전히 대등하므로, 각각 가운데에 올 가능성은 $\frac{1}{3}$"

- **Step 3**: 구체적 예시로 확인 (progressive disclosure)
  - {1, 4, 6}을 뽑은 경우:
    - 수빈=4(가운데), 지원=1, 선아=6 → 수빈이가 가운데
    - 수빈=1, 지원=4(가운데), 선아=6 → **지원이가 가운데** ★
    - 수빈=1, 지원=6, 선아=4(가운데) → 선아가 가운데
    - (나머지 3가지도 이 3그룹에 하나씩 속함)
  - "어떤 수 조합이든, 가운데 자리에 각 사람이 올 확률은 동일"

- **Step 4**: 핵심 계산
  - 서로 다른 수 120가지 중 지원이가 가운데인 경우
  - $\frac{120}{3} = 40$가지
  - 핵심 문장 강조: **"3명의 이름을 넣는 경우는 3가지, 우리가 원하는 건 1가지"**

---

**슬라이드 6: 최종 계산**

- 풀이 정리를 단계별로 표시:
  - 전체 경우의 수: $n(S) = 6^3 = 216$
  - 서로 다른 수 3개가 나오는 경우: $6 \times 5 \times 4 = 120$
  - 이 중 지원이가 가운데: $\frac{120}{3} = 40$
  - 확률: $P = \frac{40}{216} = \frac{5}{27}$
- 소수로 $\approx 0.185$ (약 18.5%)
- "약 5번 중 1번꼴로 지원이가 가운데"

---

### 슬라이드 기술 요구사항

- 단일 HTML 파일, 화살표 키(←→) 또는 화면 클릭으로 슬라이드 전환
- KaTeX로 수식 렌더링
- 말판: 1~6칸 수직선 또는 수평선, SVG 또는 CSS 격자
- 말: 색이 다른 원형 마커 (수빈=빨강, 지원=파랑, 선아=초록)
- □□□ 배열: 3칸 박스로 시각화, 이름 카드가 칸에 들어가는 느낌
- **progressive disclosure**: 각 슬라이드 내에서 클릭/키 입력마다 내용이 단계적으로 나타남
  - 특히 슬라이드 5의 6가지 배정은 하나씩 나타나야 함
- 애니메이션 최소화 — fade-in 정도만 허용, 장식적 효과 없음
- 교사가 프로젝터로 보여주는 용도: 큰 글꼴(최소 24px), 높은 대비, 여백 충분

## 10. 클로드 코드 전달 시 추가 지시

- 이 명세서와 함께 **교과서 전사본** 파일을 함께 전달합니다.
- 학습지 구조는 **Phase 1 (교과서) → Phase 2 (외부 문제)** 순서.
- 인터랙티브 요소는 해당 문제의 본문 직후, 풀이 토글 직전에 삽입.
- "수학으로 생각 넓히기" 문제는 학습지에 문제 텍스트만 포함 (인터랙티브 없음). 풀이 토글에 정답 포함.
- **슬라이드는 학습지와 별도 파일**로 생성. 학습지 HTML 1개 + 슬라이드 HTML 1개 = 총 2개 파일.
- 명세서의 정답표가 ground truth — 클로드 코드가 독립적으로 재계산하지 말 것.
